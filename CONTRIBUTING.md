# 🤝 Contributing to The Vibe Coding Playbook

Welcome! We're thrilled you want to contribute. This project exists because of people like you — people who believe everyone should be able to build things with technology, regardless of their coding background.

Whether you want to fix a typo, add a new recipe, share a success story, or improve an existing chapter — you're in the right place. Every contribution matters.

---

## How You Can Help

### 📝 Improve Existing Content

- **Fix typos or broken links** — Small fixes are always welcome!
- **Clarify confusing sections** — If something tripped you up, it's probably tripping up others too. Rewrite it so it clicks.
- **Update outdated information** — Tools and platforms change fast. If a step no longer works, help us fix it.

### 🍳 Add a New Recipe

Recipes are step-by-step guides that walk someone from zero to a finished project. If you've built something cool using vibe coding, turn it into a recipe! Great recipe ideas include:

- Web apps (dashboards, marketplaces, booking systems)
- Mobile apps (using React Native or Flutter)
- Browser extensions
- Automations and integrations (Zapier alternatives, API connectors)
- AI-powered tools (chatbots, content generators, data analyzers)

### 🧠 Contribute Prompts

The heart of this playbook is great prompts. If you've discovered prompts that work really well with AI coding tools, share them! Include:

- The exact prompt text
- Which AI tool you used it with
- What it produces
- Any tips for getting the best results

### 🌟 Add a Showcase Entry

Built something using this playbook? Share it in the showcase! Tell us:

- What you built
- Which recipes or chapters helped you
- A link to the live project (if available)
- What you learned along the way

### 💡 Suggest New Topics

Open an issue with the "suggestion" label if you have ideas for new chapters, recipes, or topics we should cover.

## The PR Process (Step by Step)

New to pull requests? No worries — here's exactly what to do:

### 1. Fork the Repository

Click the **"Fork"** button at the top of this repo. This creates your own copy.

### 2. Create a Branch

```bash
git checkout -b my-contribution
```

Name it something descriptive, like `add-recipe-booking-app` or `fix-chapter-3-typo`.

### 3. Make Your Changes

Edit or add files. Follow the style guide below.

### 4. Commit Your Changes

```bash
git add .
git commit -m "Add recipe: booking app with Calendly integration"
```

Write a clear commit message that explains what you did.

### 5. Push and Open a PR

```bash
git push origin my-contribution
```

Then go to the original repo on GitHub and click **"Compare & Pull Request."** Fill in:

- **What** you changed
- **Why** you changed it
- **Anything** the reviewer should know

### 6. Respond to Feedback

A maintainer will review your PR. They might suggest changes — that's normal and collaborative, not criticism! Make the requested changes and push again.

## Style Guide

This playbook has a very specific tone. Please follow these guidelines so everything feels like it was written by the same friendly human.

### Voice and Tone

- **Friendly and encouraging** — Write like you're helping a friend, not writing a textbook.
- **You can do this!** — Assume the reader is smart but new to coding. Never make them feel dumb.
- **Conversational** — Use contractions ("you're," "it's," "don't"). Write the way you talk.
- **Enthusiastic without being over the top** — Use exclamation points sparingly. One per paragraph max.

### Language Rules

- **No jargon without explanation** — If you must use a technical term, explain it immediately. *"A repository (basically a folder for your project on GitHub)..."*
- **Use analogies** — Compare technical concepts to everyday things. *"An API is like a waiter in a restaurant — you tell it what you want, and it brings it back from the kitchen."*
- **Short paragraphs** — 2–3 sentences max. Walls of text are intimidating.
- **Use bullet points and numbered lists** — They're easier to scan than paragraphs.
- **Active voice** — "Click the button" not "The button should be clicked."

### Formatting

- Use `##` for major sections and `###` for subsections
- Use **bold** for emphasis and `code blocks` for anything technical (file names, commands, code)
- Use `> blockquotes` for prompts the reader should copy-paste into their AI tool
- Use tables for "at a glance" information
- Use emojis sparingly to add personality (✅, 💡, 🎉, 🚀) — one per heading is plenty
- Add a "What to check" note (✅) after steps that produce visible results
- Add "Tip" callouts (💡) for helpful but non-essential advice

### Recipe Format

Every recipe should follow this structure:

1. **Title and one-line description**
2. **"At a Glance" table** — Difficulty, Time, Tools, Prerequisites, What You'll Build
3. **"Before You Start" section** — Setup steps before the main recipe
4. **Numbered steps** — Each step has a clear prompt in a blockquote and a "What to check" note
5. **"You Did It!" celebration section** — Congratulate the reader and suggest next steps
6. **Troubleshooting table** — Common problems and their fixes

## Code of Conduct Summary

We're committed to making this a welcoming community for everyone. Here's the short version:

- **Be kind** — Treat everyone with respect. We were all beginners once.
- **Be patient** — Not everyone learns at the same speed. That's okay.
- **Be inclusive** — Welcome people of all backgrounds, experience levels, and perspectives.
- **Be constructive** — When giving feedback, focus on the work, not the person. Suggest improvements, don't just criticize.
- **No harassment** — Zero tolerance for harassment, discrimination, or hostile behavior of any kind.
- **No gatekeeping** — Never mock someone for not knowing something. "There's no such thing as a stupid question" isn't just a saying here — it's a rule.

If someone violates these guidelines, they'll be asked to stop. Repeated violations will result in being removed from the community.

## Reporting Issues

### Found a Bug or Problem?

1. Go to the **Issues** tab
2. Click **"New Issue"**
3. Use a clear title like "Recipe: Landing Page — Step 5 API route returns 404"
4. Describe:
   - What you expected to happen
   - What actually happened
   - Which recipe or chapter the problem is in
   - Your setup (operating system, AI tool you're using)
5. Add the appropriate label: `bug`, `typo`, `outdated`, or `confusing`

### Have a Suggestion?

Open an issue with the `suggestion` label. Tell us what you'd like to see and why it would help.

### Experiencing a Code of Conduct Violation?

Email the maintainers directly at the address listed in the repository. All reports are treated confidentially.

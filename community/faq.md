# ❓ Frequently Asked Questions

> Got questions? We've got answers. This FAQ covers the most common things people wonder about when they first discover vibe coding. No question is too basic — we've all been there!

---

## The Basics

### 1. What even IS vibe coding?

Vibe coding is a way of building software by describing what you want in plain English (or any language!) and letting AI tools write the code for you. Instead of memorizing programming syntax, you focus on your ideas, your users, and your product — and use AI as your coding partner. The term was coined by Andrej Karpathy, who described it as a style where "you fully give in to the vibes, embrace exponentials, and forget that the code even exists."

### 2. Is vibe coding "real" programming?

Yes! The apps you build are real. They run real code, serve real users, and can make real money. The code itself is just as valid as code written by hand. Think of it this way: a book written by dictation is still a real book. What matters is the end result, not whether you typed every character yourself.

### 3. Do I need ANY coding experience to start?

Nope. Zero. Zilch. That's the whole point. If you can write a clear email or describe what you want to a friend, you have all the skills you need to get started. You'll naturally pick up coding concepts along the way, but you don't need to study them first.

### 4. What tools do I need?

At minimum, you need:
- An AI coding assistant (ChatGPT, Claude, GitHub Copilot, Cursor, Bolt, Lovable, Replit, etc.)
- A code editor (VS Code is free and most popular, or use a browser-based tool like Replit)
- A GitHub account (free — this is where your code lives)
- A deployment platform (Vercel or Netlify — both have generous free tiers)

That's it. You can start for $0.

### 5. How much does this cost?

You can start completely free. Most AI tools have free tiers, and hosting platforms like Vercel and Netlify offer generous free plans. As your app grows or your needs increase, you might spend $10-30/month on AI subscriptions and hosting. That's way less than hiring a developer.

## Building Apps

### 6. What kinds of apps can I build with vibe coding?

Honestly? Most things. Web apps, landing pages, dashboards, internal tools, booking systems, invoicing tools, SaaS products, blogs, portfolios, simple mobile apps (using web tech), browser extensions, and more. The main limitations are highly complex systems (like video games or operating systems), apps that require bleeding-edge performance, or anything that needs to pass rigorous safety/compliance audits (medical devices, self-driving cars, etc.).

### 7. Can I build a mobile app?

Yes, with some caveats. You can build a Progressive Web App (PWA) that works on phones and feels like a native app — this is the easiest path. For full native apps (on the App Store or Google Play), you'd use frameworks like React Native or Flutter, which AI tools can help with, but it's a bigger learning curve. Start with a PWA.

### 8. How long does it take to build an app?

It depends on complexity, but a simple MVP can be built in a weekend to a few weeks. The community showcase has examples of apps built in 10 days to 4 weekends. Don't try to build everything at once — start with the smallest useful version and add features over time.

### 9. What if AI generates buggy code?

It will. Regularly. And that's okay! AI-generated code has bugs for the same reasons human-written code has bugs — edge cases, misunderstandings, and complexity. The key is to:
1. Test as you build (don't write 500 lines before checking if anything works)
2. Use AI to help you fix bugs too (paste the error message and ask for help)
3. Learn to spot common patterns over time
4. Keep your app simple — fewer features = fewer bugs

### 10. What if the AI doesn't understand what I want?

Be more specific! Instead of "make a login page," try "make a login page with email and password fields, a 'Forgot Password' link below the form, a blue 'Sign In' button, and a link to the sign-up page at the bottom. Use Tailwind CSS for styling." The more details you provide, the better the result. And if it's still wrong, tell the AI what to change — think of it as a conversation, not a one-shot command.

## Business & Legal

### 11. Can I sell apps I build with vibe coding?

Absolutely, 100%, yes. The code generated by AI tools is yours. You can sell it, charge subscriptions, offer it as a service — whatever you want. Thousands of people are already running businesses on vibe-coded apps. There is no legal difference between code you wrote by hand and code generated by AI at your direction.

### 12. Is it legal to use AI-generated code?

Yes. The major AI coding tools (ChatGPT, Claude, GitHub Copilot, etc.) all have terms of service that give you ownership of the code they generate. That said, it's always smart to:
- Read the terms of service of the specific tool you're using
- Avoid asking AI to reproduce large blocks of specific copyrighted code
- Use open-source dependencies responsibly (check their licenses)

### 13. Do I need to disclose that my app was built with AI?

No, there's no legal requirement to disclose that you used AI tools to write your code, just as you wouldn't need to disclose that you used an IDE with autocomplete. The exception would be if a specific industry, client contract, or platform requires it.

### 14. Can I hire a developer to continue working on my vibe-coded app?

Yes! AI-generated code is regular code. Any developer can pick it up, read it, modify it, and extend it. It's a good idea to:
- Keep your code organized (AI can help with this too)
- Use version control (Git/GitHub) from the start
- Write a brief README explaining what the app does and how it's structured
- Consider having a developer review your code for quality before a major launch

### 15. What about data privacy and GDPR?

If your app collects user data (emails, names, payment info, etc.), you need to comply with privacy laws like GDPR (in Europe) and CCPA (in California). This means:
- Having a privacy policy
- Getting user consent for data collection
- Letting users delete their data
- Storing data securely

AI can help you draft a privacy policy and implement these features, but for a serious business, consult a lawyer. Basic privacy compliance is not as scary as it sounds.

## Security & Quality

### 16. Is my vibe-coded app secure?

AI-generated code isn't inherently more or less secure than human-written code. But as a beginner, there are common security mistakes to watch for:
- Never put API keys or passwords directly in your code (use environment variables)
- Use established authentication solutions (like Supabase Auth or NextAuth) instead of building your own
- Always validate user input on the server side, not just the client side
- Keep your dependencies updated
- Use HTTPS (free on Vercel/Netlify)

Ask AI: "Review my code for security vulnerabilities" — it's pretty good at catching common issues.

### 17. How do I handle payments?

Use Stripe. Seriously, don't try to build payment processing yourself. Stripe handles all the scary stuff (credit card security, fraud detection, compliance) and has excellent documentation. AI tools are great at helping you integrate Stripe. You'll pay about 2.9% + $0.30 per transaction, which is standard.

### 18. Will my app scale if it gets popular?

For most solopreneur apps, scaling is a "good problem to have" and modern hosting platforms handle it well. Vercel and Railway auto-scale within their plans. If you're using Supabase for your database, their free tier supports up to 50,000 monthly active users. If you genuinely outgrow these limits, you'll be making enough money to hire help.

## Tools & Workflow

### 19. Which AI tool is best for vibe coding?

There's no single "best" tool — it depends on your style:
- **Cursor / GitHub Copilot in VS Code:** Best if you want to work in a code editor and have AI assist you inline
- **Bolt / Lovable / v0:** Best for generating full UIs visually and quickly
- **ChatGPT / Claude (chat):** Best for planning, debugging, and getting explanations
- **Replit:** Best for a fully browser-based experience (no setup needed)

Most vibe coders use a combination. Try a few and see what clicks.

### 20. What if my favorite AI tool shuts down or changes pricing?

This is a valid concern, but your code is YOUR code. It lives in your GitHub repository, and you can take it to any other tool at any time. The code doesn't depend on the AI tool that generated it. To protect yourself:
- Always keep your code in GitHub (don't just build inside a proprietary tool)
- Export your data regularly
- Don't depend on any single AI tool — the prompts in this playbook work with any of them

### 21. Do I need to learn Git?

Yes, but only the basics. You need to know how to:
- Save your work (commit)
- Upload your code (push)
- Go back to a previous version if something breaks (revert)

That's about 5 commands. AI can walk you through every single one. Most code editors (like VS Code and Cursor) also have point-and-click Git buttons so you never touch the command line.

### 22. Should I learn to code "for real" eventually?

It helps, but it's not required. The more you vibe code, the more you'll naturally absorb coding concepts. You'll start recognizing patterns, understanding error messages, and even reading code. Think of it like learning a foreign language through immersion rather than textbooks. If you want to go deeper, great — but don't let "I should learn to code first" stop you from building right now.

## Common Worries

### 23. Am I "cheating" by using AI?

No more than a photographer is "cheating" by using Photoshop, or an architect is "cheating" by using CAD software. Tools evolve. The value you bring is your vision, your understanding of the problem, your users, and your creativity. The code is just the medium.

### 24. Will developers judge me?

Some might, but who cares? The best developers will respect anyone who ships a product that solves a real problem. The developer community is increasingly embracing AI tools themselves. Focus on building things that help people, not on impressing gatekeepers.

### 25. What if I break something?

You will! And that's fine. That's what Git is for — you can always go back to a previous working version. You can't permanently break your app unless you try really hard. The worst that usually happens is your deployed site shows an error page, and you roll back to the previous version (one click on Vercel/Netlify) while you fix it.

### 26. I feel overwhelmed — where do I actually start?

Start with ONE thing:
1. Pick a problem you personally have
2. Open an AI chat (ChatGPT, Claude, whatever)
3. Type: "I want to build a simple web app that [solves your problem]. I have zero coding experience. Walk me through the very first steps."
4. Follow the instructions step by step

Don't read 10 tutorials. Don't compare 15 tools. Don't plan the perfect architecture. Just start building. You'll figure out the rest as you go.

### 27. Can AI replace professional developers entirely?

Not today, and probably not anytime soon. AI is an incredible accelerator, but complex systems, critical infrastructure, and large-scale applications still need experienced engineers. What AI DOES do is make it possible for non-developers to build simple-to-moderate apps that would have previously required hiring someone. Think of it as a power tool — it doesn't replace the carpenter, but it lets more people build a bookshelf.
